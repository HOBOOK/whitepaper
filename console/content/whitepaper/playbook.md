# 아키텍처 플레이북

## API
- CRUD는 REST, 이벤트 스트리밍은 SSE(예: 텔레메트리, 작업 상태).
- 안정적 버저닝: `/api/v1/*`. 하위 호환을 유지하고 필드 추가와 페이지네이션을 우선합니다.

## 데이터
- 시계열 메트릭은 MongoDB/TS 또는 Influx 계열 스토어.
- 자산/라인/로봇에 대한 표준 ID와 스키마(MBOM/eBOM/DTOM).

## 디지털 트윈
- 시각화는 Three.js; 물리 정확도는 Isaac‑Sim/Lab.
- 씬 내보내기: GLB + 메타데이터; 시나리오 시드는 결정적입니다.

## 운영
- SLO: 읽기 99.9%, 쓰기 99.5%; 에러 버짓이 속도를 결정.
- 블루/그린 또는 카나리; 롤백은 한 번의 명령으로.
