# 플랫폼 원칙

## 1. 단일 진실 원천(Single Source of Truth)
데이터는 한 번 기록하고 어디서든 재사용합니다. 데이터 사일로를 피하고 이벤트 스트림을 채택합니다.

## 2. 기본값은 실시간
대시보드와 API는 초 단위 신선도를 목표로 설계합니다. 비동기 작업은 존재하되, 기본은 라이브입니다.

## 3. 시뮬레이션 우선
모든 변경은 배포 전에 디지털 트윈에서 검증합니다. 시나리오는 재현 가능하고 버전이 관리됩니다.

## 4. API 일관성
CRUD는 REST, 스트리밍은 SSE를 사용합니다. 멱등성과 페이지네이션은 필수입니다.

## 5. 운영 가능성(Operability)
추적, 메트릭, 로그를 일급 시민으로 다룹니다. SLO 없이 릴리스는 없습니다.
